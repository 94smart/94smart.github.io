求助：我的Firefox无法正常访问Gmail了

Google浏览器浏览器随笔随笔

昨天早上开始，我工作用的Firefox 1.5.0.6访问Ajax版Gmail出错，提示信息如下：

Loading…

This seems to be taking longer than usual.

If you are using a slow Internet connection, you can wait a bit longer for this page to finish loading, or just use basic HTML view for now.

If you are using your normal Internet connection and you usually get past this loading step without any problems, please refresh this page in your browser. If you continue to have trouble loading your account, please visit the help center for troubleshooting information.

跟着里面的链接，点到了Gmail的帮助信息，看了一下上面列的可能出现冲突的第三方软件，除了Maxthon（Maxthon访问反而正常？！）我一个都没装，根据提示这种情况可能是缓存闹得，就按提示清空Firefox的缓存（看了才知道我根本就没使用缓存，那里设置的缓存容量是0M）。

按照提示信息” 某些情况下，可能需要多次清空您的缓存，才能彻底解决这一问题”，我登出又登入了好多次，缓存也清空了无数次，而且http://mail.google.com/和https://mail.google.com/都试过了，还是不行。

昨天回家发现家里的Firefox也不能访问Gmail，真是奇怪了。

所以只好在这里求救，看看哪位达人可以拯救我，知道的请在这里留言，或者使用Gtalk联系我（94smart）~

Update：还是不知道哪里出的问题，FireBug查出Gmail的页面有一个js错误，好像是导致Gmail Loading剧卡的关键~

Update2：刚才试了一下2.0b1，居然可以用，近期准备全部切换至Firefox 2.0

Update3：今天（8-17）偶然试用Firefox 1.5访问Gmail，居然没问题了，真是恢复得莫名其妙~

升级到Wordpress 2.0.4

BlogBlogWordpressWordpress站务发展站务发展

今天终于升到了Wordpress 2.0.4，因为Wordpress发布更新公告，说有安全漏洞，呼吁使用2.0-2.0.3的用户尽快升级，最近几天看到好多人在说这个。

本来不打算升级的，嫌麻烦，不过在Dreamhoststatus看到，Dreamhost提供了一键升级的功能，嗯，不错，那就升了吧。

使用Dreamhost的升级功能，需要注意：升级前一定要备份数据库、页面的主题模板及使用的WP插件。虽然Dreamhost会把原安装目录加上.old后缀，但最好还是自己备份一下。
