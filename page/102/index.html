Zoundry升级了

BlogBlogIM

今天打开Zoundry的时候，发现它已经升级到0.9.137了，赶紧下一个来用。

官方资料里面特别提到了段落操作的改进：

New Enter key behaviour:
Pressing Enter key starts a new paragraph (

); Pressing Shift+Enter inserts a line break (
) within the same paragraph. Use Shift+Enter if you want to start a new line on the same paragraph.

按回车键开始一个新的段落（

）；按Shift+回车键是在同一个段落里面换行（
）

嘿，两周前我还给他们写过信，说这个事情，他们回信说1、2个月内解决，他们速度还是挺快的，现在这个操作方式方便多了，也更像Dreamweaver了。

另外，Zoundry对自己的不足也作了一下改进：

  • 比如自己不支持复杂HTML（比如table）的编写，就提供了OpenHtmlFile的方法，让用户可以直接打开编辑好的HTML
  • 增加了PasteAsHtml，让用户可以选择直接粘贴HTML内容，或仅粘贴文本内容
  • 为图片加链接的时候不需要鼠标拖拽成高亮状态了，直接在图片上右键点”Create Link”就行了

以上是我比较感兴趣的更新，其他更新请看这里：http://www.zoundry.com/releasenotes.html

不知道Zoundry以后能不能支持多语言包，这样就可以彻底推广了。

今日流水账 - 说说当当和豆瓣的八卦

Tag卓越哈利波特天涯当当电子商务社区网站

先说当当的，还真让laobai给说中了，当当对《哈利·波特与混血王子》搞了个差价返还，退到我的帐户￥16.5元。当当在这次哈6的价格战中好像一直处在劣势： 一，当当和卓越在哈6上市前搞的预售活动中，价格、VIP资格、附加值（优惠购买哈7等）上基本持平，具体过程我不了解，但是给人的感觉是当当一直在跟随卓越； 二，一开始卓越就说自己是首发，在首页放了一个大的倒计时，且广告中出现的都是”首发”，其实当当也是首发，但是对于这个当当根本就没做宣传，我估计让不少人都误会了，很多哈迷是不在乎在哪里买的，只在乎能不能以最快的时间看到（我也是）； 三，卓越率先降价到5折，且对于预售的用户进行差价返还，当当也只好跟着做，先打个5折，然后差价返还。 这个事情还在继续，如果接着发展下去，不知道哪家先撑不住。 再说豆瓣，昨天欣总和老冒都在说”为什么没有人抄豆瓣？”，我觉得”抄”这个事分为两种，一种是完全复制，一种是模式复制。 完全复制就是豆瓣做什么别人就做什么，比如豆瓣是做图书评论、比价起家，这样的网站现在就有，不知道好图书算不算什么算是？ 模式复制说的是抽出豆瓣的具体内容，书啦、电影啦、音乐什么的，只留下他的模式，比如评论、圈子、比价、Tag等，据我所知，今年6月的时候就有一个这样的项目，具体是哪个做的是什么，现在未经他人同意不好在这里说，但是我可以保证已经有了，请大家拭目以待。 另外，欣总和老冒都提到如果天涯抄豆瓣会怎么样，老冒说：”天涯抄袭豆瓣？ 如果他们这么干，我预言注定失败。不信？我们打赌！” 我很想跟他打一赌，不过我更看好有用户基础的社区、电子商务网站抄袭豆瓣的模式，而不是完全复制豆瓣。在这方面，天涯有足够的人气，只要稍加应用，没准真能成功。我也看好当当或卓越这样电子商务网站去复制豆瓣，不过看来与其复制还不如去收购豆瓣，为C2C添一把柴。 这篇流水帐本来应该昨天写的，今天写出来有点变味了，因为睡了一觉我已经忘记了最初的一些细节和想法，下次有感觉一定要马上记下来，好记性不如烂笔头呀~ 2005/12/6 Update：今天有人留言觉得”完全复制”那段话没有经过调查就写出来，现在已经划掉。

今日流水账 - 框框和代码

TagTag网站

前几天就看到了Zola的代码，用来防止被页面被其它网站使用FRAME调用。它的原理是发现被FRAME调用就弹出Alert，放置警告信息和CC许可，点击确定后跳转到CC许可。

发现这样有点不足，因为被FRAME调用的页面还是会显示出来，如果看的人不点击确认按钮，那个Alert的作用就不大了。

于是乎自己写了一个，原理是：发现页面被FRAME调用后，就用一个DIV把内容部分的标题和部分正文盖住，同时在DIV上面放置警告信息和CC授权以及原文链接，点击以后在原FRAME外打开一个新窗口。

这个方法也有问题就是DIV的位置需要人工设定，每个Blog的设置是不同的，这个只能自用，比较可惜，在这里就不贴代码了。

今天在Keso的Blog上看到还有更简单的办法：其实不用这么麻烦，读者要阅读的是你的内容，而不是CC许可，简单的办法就是，将被框住的内容，去掉框架后重新打开。

`<script type="text/javascript"><!-- if ( top.location !== self.location ) { top.location=self.location; } //--></script>`

唯一的缺点是对禁用Javascript的浏览器无效，不管怎么样这是我看到的最好的解决办法，在这里强烈推荐！

Update：这个方法的传播途径是维基百科=>Slashdotcn的匿名评论\=>费乐沃\=>Keso\=>我=>你，挺有意思的~
