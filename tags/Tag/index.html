2005-11-02 今日流水账 - 说说当当和豆瓣的八卦

Tag 卓越 哈利波特 天涯 当当 电子商务 社区 网站

先说当当的，还真让laobai给说中了，当当对《哈利·波特与混血王子》搞了个差价返还，退到我的帐户￥16.5元。当当在这次哈6的价格战中好像一直处在劣势： 一，当当和卓越在哈6上市前搞的预售活动中，价格、VIP资格、附加值（优惠购买哈7等）上基本持平，具体过程我不了解，但是给人的感觉是当当一直在跟随卓越； 二，一开始卓越就说自己是首发，在首页放了一个大的倒计时，且广告中出现的都是"首发"，其实当当也是首发，但是对于这个当当根本就没做宣传，我估计让不少人都误会了，很多哈迷是不在乎在哪里买的，只在乎能不能以最快的时间看到（我也是）； 三，卓越率先降价到5折，且对于预售的用户进行差价返还，当当也只好跟着做，先打个5折，然后差价返还。 这个事情还在继续，如果接着发展下去，不知道哪家先撑不住。 再说豆瓣，昨天欣总和老冒都在说"为什么没有人抄豆瓣？"，我觉得"抄"这个事分为两种，一种是完全复制，一种是模式复制。 完全复制就是豆瓣做什么别人就做什么，比如豆瓣是做图书评论、比价起家，这样的网站现在就有，不知道好图书算不算什么算是？ 模式复制说的是抽出豆瓣的具体内容，书啦、电影啦、音乐什么的，只留下他的模式，比如评论、圈子、比价、Tag等，据我所知，今年6月的时候就有一个这样的项目，具体是哪个做的是什么，现在未经他人同意不好在这里说，但是我可以保证已经有了，请大家拭目以待。 另外，欣总和老冒都提到如果天涯抄豆瓣会怎么样，老冒说：“天涯抄袭豆瓣？ 如果他们这么干，我预言注定失败。不信？我们打赌！” 我很想跟他打一赌，不过我更看好有用户基础的社区、电子商务网站抄袭豆瓣的模式，而不是完全复制豆瓣。在这方面，天涯有足够的人气，只要稍加应用，没准真能成功。我也看好当当或卓越这样电子商务网站去复制豆瓣，不过看来与其复制还不如去收购豆瓣，为C2C添一把柴。 这篇流水帐本来应该昨天写的，今天写出来有点变味了，因为睡了一觉我已经忘记了最初的一些细节和想法，下次有感觉一定要马上记下来，好记性不如烂笔头呀~ 2005/12/6 Update：今天有人留言觉得"完全复制"那段话没有经过调查就写出来，现在已经划掉。

2005-10-29 今日流水账 - 框框和代码

Tag Tag 网站

前几天就看到了Zola的代码，用来防止被页面被其它网站使用FRAME调用。它的原理是发现被FRAME调用就弹出Alert，放置警告信息和CC许可，点击确定后跳转到CC许可。

发现这样有点不足，因为被FRAME调用的页面还是会显示出来，如果看的人不点击确认按钮，那个Alert的作用就不大了。

于是乎自己写了一个，原理是：发现页面被FRAME调用后，就用一个DIV把内容部分的标题和部分正文盖住，同时在DIV上面放置警告信息和CC授权以及原文链接，点击以后在原FRAME外打开一个新窗口。

2005-10-12 Zoundry和Drupal配合的使用方法

api IM Tag Tag

经过两天的使用，我简单总结一下Zoundry和Drupal配合的使用方法。

  1. 首先设置Drupal的Blogapi，路经是/admin/settings/blogapi，类型为MovableType。
    我试了一下MovableType类型的支持最好，可以从Zoundry里面直接调用Drupal的分类信息。
  2. 接着设置Zoundry的Blog帐号，在Tools菜单下，它提供一个向导，可以轻松完成设置。

    需要注意的是，要在类型里面选择Movable Type，然后在XML-RPC URL框里面填上你的xmlrpc地址，Drupal系统的在/xmlrpc.php，完整的是这个样的：http://www.your.com/xmlrpc.php
  3. 从Zoundry发布Blog，还可以做到自动上传正文图片，方法是设定Media Repository，也在Tools菜单下，设定如图：

    然后就可以从自己的硬盘里选择图片，插入到文章里面，同时可以调整缩略图大小，发布内容的时候图片（包括缩略图和原图）就会传到指定的地址。

电视这个东西基本上已经从我的视线中消失了，但是总会在一些特定场合出现，比如去别人家吃饭，人家喜欢一边吃饭一边看电视……在这样的场合下我又不得不看，关键是饭点儿能有什么节目看呢？正赶上人家喜欢看BTV7的《生活面对面》。 这个节目很2.0，为什么这么说呢？ 1、这个节目说的是日常生活中的事情，看这个节目的人都是老百姓； 2、这里面的片源全部是观众提供的，而且都是观众用自己的DV拍摄的，电视台拿来稍微处理一下就播放了； 3、这个节目还有反馈机制，就是通过短信等来给这些片源的作者投票，支持他们的作品。 简单说，这个节目除了主持人的引导之外，已经都是观众自己的信息采集、传播、消化了，这个节目已经变成了一个平台，相信这样的节目收视率很有保障，而且这样的节目还会越来越多（例如超女）。 这些2.0的电视节目其实在为Podcast做准备，培养了节目制作者和观众群，如果改变节目的形态（数字化），可以很方便转换到其他平台，例如土豆网等Podcast网站。 想想看这种节目的操作手法与土豆网的是否类似： ·电视节目平台 VS 土豆网 ·制作者将DV带寄给节目组 VS 播客将自己的节目上传到土豆网 ·观众通过电视观看节目 VS 用户在土豆网观看节目 ·观众通过短信等反馈 VS 用户通过留言等反馈 ·电视台获得收视率 VS 土豆网获得访问量 ·制作者和播客都得到其他人的支持，有动力不断完善，形成良性循环 …… 很像吧，当然后面就是，电视节目和土豆网都会得到利益回报，如果他们会和节目制作者分享利益，他们都会有好的发展，但是就实际情况来说，土豆网等Podcast网站在这方面显然比电视有优势。 Podcast的发展很有可能动摇电视的权威性，从而最终实现“去中心”。这里说的“去中心”并不是说电视台就都没了，不存在了，说的是他们可能会换另外一种形态去存在（进化？），可能就把电视节目都搬到网上，和土豆网们分享Podcast市场，比如以后可能会出个生活面对面网站用机器处理播客们的节目，也可能会有个超女在线PK台（不是Sohu那个）等等，IPTV为此将获得大批收入（还是盛大有前瞻性）。 结局如何就看造化了~

2005-08-09 今天才知道，原来WAP这么不安全

Tag Tag

用手机上WAP站的注意了，如果你使用中国移动（联通的CDMA我没有测试）的WAP服务，你的手机号将被WAP程序轻易获得，这也解释了为什么会有那么多SP的PUSH广告信息骚扰、以及莫名其妙的服务订阅。 因为中国移动（包括全球通、动感地带及神州行）的WAP网关会把机主的信息一起发给WAP站，其中包括以明码发送机主的手机号。 接着SP就可以给该手机发送PUSH信息，如果你误点了“是”，就不幸的订阅了某个服务。据说有的SP就是这样赚了很多钱，就算你去退订，如果超过一定天数你还是必须支付这个服务的费用。 不信邪的可以用手机访问：wap.94smart.com，你将会看到你的手机号。 Update：好像很早就有了，是我孤陋寡闻今天才知道。

2005-06-10 羊群效应和SNS

IM sns Tag Tag

羊群效应的一个实例： 办公室里，一个同事站在窗户前说：“哦，太真NB了！”马上（不到一分钟）就有一堆人拥到窗前争着去看，不管窗外是否真的有吸引人的事情，听到的人都有想去看的冲动。如果是真的，看到的人会扩大其影响，如果是假的，对大家也没什么损失，隔段时间还是会受人影响再到窗前去看。 另一个实例： 一个女孩在座位上笑得前仰后合的，周围的人都围过去看，她自己说道：“看他的MSN Space，太好笑了~”对于一向喜欢凑热闹的人来说，MSN Space的小黄花太有价值了，可以提醒好友自己的Space更新了。就算不是为了博美女一笑，因为MSN上的联系人都在用，自己也不能落后，于是乎MSN Space已成为MSN用户的首选Blog服务。 只要有人群的地方就有会羊群效应，有羊群效应的地方就可以发展社会性网络服务（以下称SNS）。 我认为SNS的发展需要依靠原有的人际网络。一个圈子里面的一员如果对某件事物产生兴趣，经过他自己的传播，可能很快就会影响整个圈子。 现在的SNS的发展路线反而是相反的，非要让用户先在自己的系统里搭建人际网络，然后再发挥SNS的优势，这样白手起家的做法怎么可能斗得过QQ或者MSN呢？这也是我看好MSN Space和Yahoo 360的其中一个原因。 所以经营SNS的兄台们需要考虑的是，怎么从现有的人际网络出发，用自己的服务自然的将其他网络的用户接入自己的系统。 今天终于胡说完了，不同意的就拍我~
