Web2.0，培训你的用户吧

TagTag用户体验

其实不管是不是Web2.0，网络应用/服务也应该以用户体验为中心，虽然"用户永远都是对的"这句话很别扭，但这其实是所有服务都应该去遵守、去实现的。

我对用户体验的理解是，用户爱怎么用就怎么用，怎么顺手就怎么用。但这种放纵的用法可能并不能体现你网络应用的特点，服务及其有可能被滥用（例如网摘和Tag），所以，对于网络应用和服务来说，最基本的一条是要告诉用户你都可以怎么用，以及你有怎么样的自由度，这是在本文里培训的简单定义。

当你对用户做到足够培训的时候，用户就不会问"去哪里找……？"、"……怎么操作？“等等问题了，这样普通用户将会逐渐变成你的忠实用户。

这方面，Flickr和Del.icio.us做得很好，包括详细的帮助信息，友好的UI，还有在合适的位置提供技巧、提示。他们的做法已经成为其他人抄袭复制的对象，我赞成这样的抄袭复制，从培训用户的角度，越通用的用户体验越能使用户快速掌握应用，从而更容易的让用户喜欢上你的产品。

当然，不会培训用户的应用/服务也有的是，在此就不列举了。

OK，不管你的网络应用/服务是不是Web2.0的，快培训你的用户吧。

本周流水账 - 懒人一周

GoogleTagTag互联网

俗话说春困，秋乏，夏打盹，睡不醒的冬三月，到了冬天人也变得懒起来了，我的意思是懒得写Blog，当然我就是其中之一。本周基本没写，主要就是懒，其实每天都可以写两句，但是又觉得就写一两句有点像糊弄事，所以就汇总一下，写一篇本周流水账好了。

商业周刊，Web2.0

周一等车的时候买了份11月的《商业周刊/中文版》，本来是图新鲜，因为还没看过商业周刊所以买来看看，还真撞上了感兴趣的内容–Web2.0。本期以"全新的网络"为题作了一个专题，提到了一堆Web2.0公司和服务，有熟悉的Flickr、Del.icio.us、37Signals、MySpace等，还有不熟悉的SoicalText、PostSecret、SaleForces.com、Pandora等，关于内容（PostSecret）、电子社区（Cyworld）、网页书签（Del.icio.us）、AJAX等还有专门的文章介绍。

社区，BBS

周三Donews的老大liuren在《Blog、MSN都是BBS》里说道：“BBS是互联网最基础的内容管理系统。直到今天它还是基础，但它的用户正在不断地被分流。“想了想，大多数网上交流方式确实可以归结为BBS或者BBS的特定形式，BlogBus的改版就是最及时的例证，博贴就算是一个分类信息状的BBS吧。

p.s.最近因为工作的关系，要做很多社区方面的事情，周二的时候写了一篇《社区相关 - 社区的信任问题》，不过发现好像没人感兴趣，本来还希望有人愿意交流社区方面的经验，看来下次这样的东西还是找论坛发吧~

Google

周五收到Google Adsense寄给我的个人识别号码，好像都申请了几个月，也早就不在乎了，不过头一次收到Google的信很兴奋，Google的信真是简单，简单到没有信封，就一张纸对折一下，然后贴胶纸封上，信的内容也很简单，有半张纸只放了一个Google的Logo，对于这样的信，不知道该说是节俭还是浪费。

FireFox

FireFox1.5终于出了，用WebLeOn介绍的方法做成PortableFirefox，真是越用越爱用，也在考虑要不要把Maxthon换掉，毕竟现在对Maxthon的留恋仅在于它可以很方便的打开和关闭标签页。

Del.icio.us

美味书签最近发布了FireFox的扩展，装上用了用，感觉不爽。看，Jan还专门写了一篇《del.icio.us Firefox Extension總是走不出框框》，真是说到我心里去了，扩展如果只是调用链接的话确实觉得很多余，如果不能实时同步，就连另一个扩展Foxylicious也比不上了。

-——–下面是废话———-

看吧，每天就一点儿确实有点寒酸，但是一周合起来又觉得乱、杂，没办法，这就是生活~

今日流水账 - 框框和代码

TagTag网站

前几天就看到了Zola的代码，用来防止被页面被其它网站使用FRAME调用。它的原理是发现被FRAME调用就弹出Alert，放置警告信息和CC许可，点击确定后跳转到CC许可。

发现这样有点不足，因为被FRAME调用的页面还是会显示出来，如果看的人不点击确认按钮，那个Alert的作用就不大了。

于是乎自己写了一个，原理是：发现页面被FRAME调用后，就用一个DIV把内容部分的标题和部分正文盖住，同时在DIV上面放置警告信息和CC授权以及原文链接，点击以后在原FRAME外打开一个新窗口。

这个方法也有问题就是DIV的位置需要人工设定，每个Blog的设置是不同的，这个只能自用，比较可惜，在这里就不贴代码了。

今天在Keso的Blog上看到还有更简单的办法：其实不用这么麻烦，读者要阅读的是你的内容，而不是CC许可，简单的办法就是，将被框住的内容，去掉框架后重新打开。

`<script type="text/javascript"><!-- if ( top.location !== self.location ) { top.location=self.location; } //--></script>`

唯一的缺点是对禁用Javascript的浏览器无效，不管怎么样这是我看到的最好的解决办法，在这里强烈推荐！

Update：这个方法的传播途径是维基百科=>Slashdotcn的匿名评论=>费乐沃=>Keso=>我=>你，挺有意思的~
