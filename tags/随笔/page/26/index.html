Ubuntu传播模式浅析

先介绍一下，Ubuntu是一种Linux的发行版，包含了16000多种软件，将桌面需要的核心软件集成在一张CD盘上，同时适用于桌面系统和服务器。

获得Ubuntu的方法很多，其中一种是免费邮寄光盘，需要说明的是，只要写明目的，Ubuntu是不会限制邮寄数量的。还有，可以从已经收到光盘的人（尤其是多索取光盘的人）那里获得，自己觉得好用才会送给朋友用。

接着来谈谈我对这种传播模式的看法：

首先，感觉Ubuntu的这种传播方式很不便宜，每个用户可以不限制数量的获得Ubuntu光盘，这就需要一大笔行销费用，还有因为是免费邮寄，不只是光盘的成本，还要承担邮寄的费用。

其次，这是病毒营销的又一个例子，上一个应该是Gmail的邀请，Ubuntu为什么不限制免费邮寄的数量？不是因为他们钱多烧的，而是他们知道口碑传播会形成良性循环，送得越多，传播也就越广。

再次，这是一次文化融入的营销，它将自由软件的精神通过实际行动得到广泛传播。

所以我最佩服的是它的这条说明：

Ubuntu完全忠于自由开源软件的发展，我们鼓励人们使用自由开源软件，改进它和继续传递它。（中文解释来自Ubuntu中文）
英文原话：Ubuntu is entirely committed to the principles of free software development; we encourage people to use free and open source software, improve it and pass it on.

最后，这种模式在国内可能不会成功，原因就不多说了。

ok，来个煞尾，我没有技术去改进它，所以我要继续传递它，Ubuntu，一个不错的Linux系统。

对RSS的粗浅理解

最近几天，RSS相关话题在逐步升温，歪脖·坑、旧烟斗这2位都写了一系列关于RSS的文章，有兴趣的朋友可以去看看。

看得起劲，所以也想写写我对RSS的粗浅理解：

  1. RSS已经替代了Feed，成为包括RSS、RDF、ATOM等这一类文件的代名词，看来这个名头是撼不动了。
  2. RSS对于Blog的传播有促进作用，信息成瘾的人会订阅一大堆RSS，然后看完所有更新的内容，他们的口号是"不求最精，但求最全"，我好像就是。
  3. RSS订阅的总量是增加的，我的原则是只要有好的内容看就订，从去年1月的1个（Keso的RSS）到今年1月的200多，我一年的增长率就**20000%**了，多可怕。
  4. RSS是社会性的，如订阅数多的RSS会被更多人订，好像大家都想知道别人订的到底好在哪里。
  5. RSS并不能带来阅读的效率，这是真的，而且随着订阅数量的增加阅读效率也会随着降低，所以如果想节省时间的话，直接订Keso网摘的RSS好了~
  6. RSS会收窄你的视线，坏处是把你局限在一个小领域。

先写这么多，回头再补~

**Update：**真是惭愧，第三条写成了200%了。

今日流水账 - 倒时差

搬来Dreamhost还没几天，感觉还不错，不过因为地域等关系总会出现一些以前没想到的事情。昨天就说过Dreamhost阻止了PHP的远程读取选项，提供另一种解决方法，今天又遇上了时差问题。

今天早上10点看我的Daily.94smart.com的时候，发现内容全是空的，真奇怪，昨天还是好好的。第一感觉是365key当了，但是马上就发现是Daily程序的问题，测了半天，一直没有找到问题。

看了一下这两天的网摘才想起来，1月2号那天没做网摘，我的Daily会把前一天的网摘显示出来，但是今天的为什么没有内容呢，昨天明明摘了不少？

后来才发现，今天虽然是1月4号，但是服务器时间（-8）为1月3号，再往前一天又没做网摘，所以内容才不显示出来，因为程序原来运行在国内（+8）的服务器上，所以没有进行时区设置，搬到国外服务器当然就栽了。

没想到这个小程序被迫出国，还得倒时差，一差还就差到16个小时~
