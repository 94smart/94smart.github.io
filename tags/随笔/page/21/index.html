貌似Sogou输入法还不错，但我还是用不惯

上周在大批Blogger推荐的情况下，我终于使用了Sogou拼音输入法。 我感觉优点在于新鲜的词、热词一打就出，速度还特别快，而且支持多种键位用法，看来他们是下定决心要挖其他输入法的墙角了。 因为我用的输入法比较单一，只用微软拼音2003，不知道它和其他哪个输入法比较像，只知道它和微软拼音操作不同。 所以它还是不适合我，原因如下：

  1. 输入拼音后按空格或数字选词后就直接打出来了，不能像微软拼音一样需要用回车确认。这一点可能别人认为是优点，而我认为是缺点。
  2. 微软拼音支持拼音的自动拆分，Sogou不支持。比如打"xian"，Sogou候选词里面只有字，而没有词，事实上我想打的是"西安"，要想达到这种目的必须输入"xi’an"。
  3. 输入法的显示条怎么隐藏？我到现在也没用出来。

还有几个小地方，就不一一列举了，我觉得，像我一样习惯微软拼音的人可能很难接受Sogou。 但是我还是推荐使用紫光拼音和拼音加加的朋友试试Sogou拼音，没准你会喜欢~ 附我的想法变化： 第一印象：Sogou也可以做拼音输入法？ 第二印象：Sogou的拼音输入法还不错 现在：Sogou的拼音输入法不适合我 p.s.经KESO一提，想起来以前还在学校里做过智能狂拼II的测试工作，起先感觉还不错，后来发现其与Windows平台上的程序兼容性极差，才知道这种与系统相关性极高的东西只能用微软的。。。。。 Update：下午刚写完这篇，晚上就看到Sogou输入法将继承Sogou Toolbar的流氓软件身份，在未来的日子里安插在各位的PC上。对于这篇blog，我不知如何是好，就留着吧，懒得删了。 Update 2：今天（6月18日）才注意到新浪的首页有Sogou拼音的广告，其中有个链接还连到我这里 Technorati : IME, Sogou

Opera Mini 2.0，用手机上网

最近在用Opera Mini 2.0手机上网，我指的是HTML网站，感觉不错。然后觉得WAP浏览变得没意义了，至少对我来说是这样。

用手机的HTML浏览器上网太慢，而且费用巨高，想想每个网页包括图片小的100多K，大的快1M，用手机上实在是太不值了。

用WAP呢，图片很少，主要是链接，新的WAP2.0更支持XHTML，速度和内容都有保证，但是毕竟是一个缩水的协议，WAP有太多不支持的功能（cookie等），感觉永远都不如HTML浏览。

现在好了，Opera Mini 2.0来了，我给自己的Nokia 6230i装了一个，JAVA的，97k，直接上HTML，速度飞快。

因为Opera Mini 2.0把页面重新渲染了，将内容按照手机的屏幕宽度顺序排列，同时只下载页面和部分图片、样式表，在小尺寸下载的情况下保证内容完整，例如Sohu首页下载完46K（为什么用sohu？！因为它!%&^%*#%^*&）。

前一阵一直用百度的WAP，因为百度可以将搜索结果处理成WAP方式的，现在不用了，可以直接上Google，爽~

以WAP的流量换来HTML的浏览，所以我前面说WAP的浏览方式已经没意义了，想试试吗？到http://mini.opera.com看看吧。

p.s.以前做网站用XHTML标准，是为方便移植，现在只要用标准制作就可以了，不用再考虑转换到WAP的问题，因为转换这个工作已经交给Opera之类的浏览器去做了，手机以后可以直接浏览HTML/XHTML。

还好Gmail没被黑

之前一度以为自己的Gmail被黑了，然后用来给我的联系人发垃圾信，今天Gmail小组给我回信我才知道只是虚惊一场。

以前曾有人说过我的信箱被报告为垃圾信箱，我没在意，前一阵只说在Gtalk上又跟我说，收到我的垃圾信，还是黄色的，我才知道是因为有人盗用的Gmail发垃圾信，收到的人就"Report Spam"，所以别人收到我的信都会被送到垃圾箱里。

于是赶紧到Gmail邮件中心去提交信息，多谢只说提供的垃圾信样本，Gmail小组已经帮我调查了，初步确定不是被黑，只是伪装我的Gmail发信，发信服务器不是Gmail的，参看Gmail小组给我回的信：

非常感谢您向我们报告。对此可能给您带来的任何不便，我们深表歉意。

您所提及的邮件不是由 Gmail 或 Google 发出的。它似乎是由某人伪造 Gmail 地址发出的，因此会被误认为是来自 Gmail。这种行为通常称为"邮件欺骗"(spoofing)。

我们非常关注这类行为。我们已经将您提供的信息转发给相关部门，由他们进行调查。

如果您认为自己的 Gmail 帐户受到了损害，或者您要报告其他欺诈行为，请单击任一 Gmail 帮助中心页左侧的"与我们联系"。

如果您认为自己可能会因身份遭窃取而受到侵害，最好与当地政府联系。

非常感谢您的理解。

Google 小组敬上

谢天谢地，还我紧张了好几天，因为现在Gmail已经成为我的主要信箱，很多重要的信都是Gmail来保管，如果真的被黑损失就太大了。

p.s.这次也反映出Gmail小组或者说Google客服的反应非常之快，联系信件都在一天内就发给我了，国内的网站客服有这么快的处理速度吗？

接着讲Flickr的八卦

去年七月份因为Flickr的一次当机，了解到他们的程序使用PHP+PEAR开发，还小小的感慨一下。

前两天，顺着DBA的Flickr 的开发者的 Web 应用优化技巧，找到了Cal Henderson写的Serving JavaScript Fast，又让我窥探到Flickr的些些技术内幕，Flickr用的模板居然是SMARTY，这个之前普遍被人诟病的模板系统。

不是说SMARTY不好，而是太好了，更像一种新的语言，我记得当时用PHP的普遍不喜欢语法复杂的SMARTY，嘿嘿，虽然技术不行但我也是其中之一，到现在还在用PHP Template，感觉好像是挺落后的。

不过，通过这篇文章同时也看到，Flickr技术大牛跟我这样的菜鸟有着巨巨巨巨巨……大的差别，就是他们用适合项目的技术，自己去优化程序语言的执行效率，我用对自己来说是简单的代码，效率问题只能听天由命了。

我觉得这好像也从侧面验证了Yahoo收购Flickr的必然，PHP之父在Yahoo，所以Yahoo有经过特别优化的PHP引擎，Flickr人用自己优化的服务器、脚本，同时使用的也是PHP，不知道这样说合适不合适，PHP的应用导致Yahoo一定收购技术相类似的、可以方便接入的Flickr。
