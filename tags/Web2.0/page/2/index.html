2006-06-20 再说E准，想到哪儿说到哪儿

之前小小推荐了一下E准，最近又用了用，想写一点补充的想法。

首先，我感觉这次测试版的推广效果还不错，正好赶上世界杯这个热潮，时间刚刚好，有那种顺水推舟的感觉。

其次，前几天看到詹老师写E准电视社区的文章，社区是发展趋势，我完全同意。但是，我不认为WAP和3G应用会是E准的主要方向，觉得E准瞄准的方向应该是电视、IPTV，或者说最起码应该是机顶盒、视频点播类的应用。

试想一下，如果E准可以增加（或合作伙伴提供）节目的点播功能，在E准电视社区的基础上，结合用户推荐等机制，其他用户不止会对推荐节目产生好印象，还可以直接浏览，从而增加该节目的收视率。

2005-07-02 尝试Feed在线阅读器

Feed IM Web2.0 Web2.0

在用过几个在线阅读器以后，我决定还是使用Bloglines，没别的，就因为它的读取速度还能令我满意，使用上还是最顺手。 最近1个月才开始用Bloglines，其实我需要的只是它的订阅查询功能，看看什么人订阅了我的Feed。其次阅读内容很方便，另外有些功能挺好的，比如共享收藏、摘录Blog、OPML的导入/导出、天气预报订阅等等。总之是好处多多，不再赘述，强力推荐一下！ Rojo看上去也还可以，不过UI做得不是很好（主要是导航混乱），读取速度有点慢（可能因为Rojo主要面向的是欧美用户），还有就是我在Bloglines里面可以正常导入的OPML，在Rojo里面无法导入。但是Rojo有比Bloglines做得好的地方，就是为Feed引入了Tag，方便了Feed的再聚合，这点需要Bloglines学习一下。 Newsgator用得不多，因为读取速度不是很快，而且显示的风格和Rojo差不太多。有一点还是不错的，就是Newsgator的为文章增加Ratings功能，这个应该算是首创，我倒觉得这是在线阅读器应该具有的另一个功能，有助于标注信息的重要性，最终达到信息的筛选。 Litefeeds是WebLeon推荐的在线阅读器，Web方面可成为Bloglines的临时替代者，下次出问题可以将就用Litefeeds。其实它的特色还在于支持手机、PDA等移动设备，我只用手机测试过，感觉有点慢，不知道2.5G时代的手机阅读是不是个好方向。 自从Bloglines广受欢迎以后就变成了被克隆的对象，国内的有Topim和Boyue两家学得挺像的。Topim是神似的简化版Bloglines，Boyue除了Logo还没变之外，UI已经尽可能的Copy Bloglines了。导入OPML这个功能上Topim的速度可以和Bloglines媲美，而Boyue的处理速度慢得实在不能令人恭维。其他方面，Boyue还支持Feed黄页、Tag和Podcast播放，Topim的功能相对有点少。 国内还有几个在线阅读器听过没用过，也就不在这里胡扯了。 最后一句话，如果在线阅读器具有Bloglines的所有实用功能，再加上Tag、Ratings等信息筛选功能的话，可能就比较好用了，而且也比较有价值。

2005-06-15 Blog圈最近出来的小玩意

api Feed IM Web2.0 Web2.0 用户体验 网站

Blog圈最近几天出了两个小玩意，网易部落的邮件发布服务和Blogcn的Rabo，都是远程发布Blog的服务。 两个东东我都试用过了，网易部落的邮件发布操作比较简单，只要用注册部落的email发信到指定邮箱就可以发布内容了，超简单，而且速度很快，支持HTML发布且可以发布图片。这个服务类似于moblog的操作方法，我估计在手机上一样可以发布内容到网易部落。 另一个，Blogcn的Rabo可就很不令人满意了。首先要下载、安装、登录，登录上去一看（Feed阅读居然要先登录？），界面主次不分，发布的时候居然弹出一个错误“保存失败”，然后就退到了主界面，刚才的内容也就这么没了。还有，我以前就是Blogcn的用户，可是登上rabo却看不到我以前的内容，这也太说不过去了。于是乎我二话不说就删了，以后也不打算用了。 两者的区别看上去在于是邮件发布还是客户端发布，如果发信使用outlook/foxmail的程序，那就没什么太大区别了，除了rabo可以查看以往内容之外，在操作上简直一无是处。 建议Blogcn反省一下，费了那么大劲，把页面上的功能倒腾到一个不是所有人都习惯的、只能用在一个网站服务上的程序，得来的用户体验会好到哪里？它会比用熟悉的邮件客户端、简单的操作、快速的发布得来的体验要好吗？ p.s.Robo跟Newz Crawler（举例而已，还有N多类似的）简直没法比，同样都是Feed阅读器+Blog客户端，但是Newz Crawler还可以设定多个帐号发布到多个Blog，支持多种API（至少三种），高下立分。
