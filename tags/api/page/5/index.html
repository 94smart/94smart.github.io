Zoundry新版本支持MSN Space

apiIMTagTag

昨天刚在《国内的BSP还在等什么？》写过，Zoundry要发布新版软件支持MSN Spaces的API，没想到今天新版就出来了，看他的更新说明：

This release contains some minor enhancements as well as bug fixes.

  1. Added official support for MSN Spaces. To use MSN Spaces, choose MSN Spaces for account type.
  2. Fixed create link to existing post bug where the currently selected text is over written with the post title.

用了一下，并不是想象中那样的更新，Zoundry只在新建Blog帐号的支持系统项目里增加了MSN Spaces，且不需要像Dareo在另外两个Blog客户端里演示使用那么麻烦，都不用填写XMLRPC地址、SSL和端口。

Zoundry的反应速度真是挺快的，不过感觉这个软件的发展方向并不是像Blogjet那种开放型的，是一个相对封闭的，主要是因为没有开放SSL、端口等高级选项，但是对现有特定服务的支持却很好，尤其他这种对高级选项的封装对最终用户有很好的可操作性。

OK，以后又可以多一个Blog可以通过Zoundry更新了，哪个BSP是下一个？

国内的BSP还在等什么？

apiTagTag用户体验

昨天从Vazi那里找到了Dareo的Space，知道MSN Spaces的MetaWeblog API已经正式开放，而且还看到支持的程序里面有Zoundry，然后怀着激动的心情用Zoundry去连，结果以失败告终。

原因是这个API默认使用HTTPS连接，而我在Zoundry里遍寻不着这个选项。最后只好去Dare那里问，用我蹩脚的英文问如何使用Zoundry连接这个API，并且表示Zoundry现在没有HTTPS和端口的选择。

今天再去的时候已经看到了Dare给我的回复：

Zoundry is one of the tools that will support the API in its next version.

同时看到Zoundry工作人员Eric Wittmann的留言：

Also, a new version of Zoundry Blog Writer 0.9 will be available shortly. The new version will have MSN spaces support. Expect it in the next couple of days (or less). Just doing some QA on it atm.

很久没更新Space了，主要是已经用惯了Zoundry，现在懒得在页面上写东西，看来在不久的将来可以用Zoundry来更新我的Space，期待ing~

在开放了部分API和PowerToys之后，我觉得MSN Space已经不亚于Blogger.com了，自由度已经很高了。另外，国外BSP厂商的开放策略对促进用户增长和改进用户体验有很大帮助。

可是国内的BSP呢？他们还在等什么？等着微软们收拾他们吗？

多少次事实证明，当微软进入某领域后，不思进取的公司将被市场和用户放弃。Blog、BSP也是这种情况，如果让微软腾出手来，估计国内的BSP服务商会死一片。

我觉得开放Blog API会是一个开端，一场血雨腥风即将到来……
