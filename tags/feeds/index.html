参加微软的网络阅读行为访谈

被拍了，再前一个星期六的上午，在朝外的良木缘咖啡馆，当时我正通过免费的无线网连接Bloglines，在前面有个数码录音，后面有个DC摄像，不断记录我的各种操作及操作的原因、想法。

这是我的又一次线上转线下的活动，微软研究院的吕昊通过Gmail找到我，希望我能参加他们的RSS阅读访谈，我们在Gtalk上沟通，最后在Google Calendar上安排好了时间和地点，有一点很奇怪，为什么不用MSN呢？

起先并不想去，但是他们的那句：“你什么时间方便，就安排在什么时间，在你能用RSS Reader的地方”，很有点那个立志用曲别针换房子的青年的意味，所以最后还是去了。

起先以为他们是要给新的阅读器产品做用户分析，后来才发现他们关注的其实并不局限于RSS阅读器、浏览器之类的用户行为，他们更在乎通过各功能的操作分析用户的思想，了解用户的阅读行为模式，最后总结出通用的用户行为，可能应用在商业软件的设计中。

参加这个访谈还有一个好处，就是发现自己很多选择上的动机，也使我开始去思考自己的行为模式，这在访谈之前是没有想象的。

最后，我也来帮忙推广一下，如果你想了解一下自己的阅读行为，想为RSS阅读研究出力，而且不太讨厌微软，可以参加他们的RSS阅读习惯调查，中文点这里，英文点那里。

慢讯 - Feedburner结盟TypePad

来自TechCrunch的消息，Feedburner于美国时间本周二宣布结盟TypePad，之后将开始实质性合作。

从此TypePad提供Blog的Feed将自动转换为Feedburner的地址，同时由Feedburner提供统计服务，不过原Feed订阅用户需要修改Feed订阅地址，好像操作有点麻烦。

FEED43，没有Feed创造Feed也要订

现在大部分空闲时间都在读Blog，这正好也是自己写得少的一个借口（或者说原因），另一个借口是懒得写，但是今天（应该是前几天）想把最近的少少经验写一写。

首先要说明的是，我花费时间阅读的顺序由高到低是网站浏览、阅读器的Feed、Gmail邮件订阅，产生网摘（或收藏）的顺序是阅读器的Feed、网站浏览、Gmail邮件订阅。

也就是说，我花费了很多时间在网站浏览，收效还不如Feed阅读，我总结这种情况的出现主要是因为不得不去看一些不支持Feed的网站，但是这些内容的过滤并不比过滤Feed和Gmail容易，因为多数只有一个标题，没有摘要可以预览，所以都是一篇篇的打开看，这样的效率确实挺低的。

所以我急需一个将网站页面内容抓取然后Feed化的工具，正好FEED43就是。

扯了这么久终于可以进入正题了，先看看FEED43首页宣传用的图：

看了图，基本上不用解释，大家就知道FEED43是用来干什么的了吧，我就是看到这张图才决定要用的。不过我还得友情来个提醒，不要被这张图骗了（？！），因为实际并不是如图那样可视化操作的，而是用输入框操作的，且对HTML代码还得有一定了解。

简述一下过程：

  1. 先输入目标页面链接及其编码，会返回该页面的HTML源码
  2. 划定范围，找到内容区域的起始及结束代码，并插入通配符{%}，比如{%}
    确定重复出现内容的格式，并为将各内容文字替换为通配符{%}，比如{%}{%}
    FEED43会根据设定去分析HTML，完成后会得的一系列item，每个item会有对应的{%N}字符串，比如{%1}=http://xxx.com，{%2}=title，{%3}=text
  3. 接着设定Feed的标题、网页链接、描述
    然后设定Feed项目模板，包括标题、链接、内容/摘要，与之前的{%N}相对应。
    这里还有一个选项可以将所有item合并到同一个item里面，结果有点像我用365key生成的昨日收集
  4. 你将得到一个数字文件名的Feed地址，接着可以修改文件名、设定Feed修改密码、设定Feed为你私有等。

嘿，我用这个工具搞定了FTchinese。

正所谓，有Feed要订，没有Feed创造Feed也要订~

如果你也有跟我一样的烦恼可以去试试这个FEED43。

p.s.如果有Bookmarklet之类的工具就更好了，看到什么就截下来然后做成Feed。

对RSS的粗浅理解

最近几天，RSS相关话题在逐步升温，歪脖·坑、旧烟斗这2位都写了一系列关于RSS的文章，有兴趣的朋友可以去看看。

看得起劲，所以也想写写我对RSS的粗浅理解：

  1. RSS已经替代了Feed，成为包括RSS、RDF、ATOM等这一类文件的代名词，看来这个名头是撼不动了。
  2. RSS对于Blog的传播有促进作用，信息成瘾的人会订阅一大堆RSS，然后看完所有更新的内容，他们的口号是"不求最精，但求最全"，我好像就是。
  3. RSS订阅的总量是增加的，我的原则是只要有好的内容看就订，从去年1月的1个（Keso的RSS）到今年1月的200多，我一年的增长率就**20000%**了，多可怕。
  4. RSS是社会性的，如订阅数多的RSS会被更多人订，好像大家都想知道别人订的到底好在哪里。
  5. RSS并不能带来阅读的效率，这是真的，而且随着订阅数量的增加阅读效率也会随着降低，所以如果想节省时间的话，直接订Keso网摘的RSS好了~
  6. RSS会收窄你的视线，坏处是把你局限在一个小领域。

先写这么多，回头再补~

对W3C网站标准的疑问

网站标准确实是个好东西，用它做的东西很高效，XHTML来写结构、CSS来写样式，样式确实和内容分开了，但是却抛掉了一部分灵活性。 首先，原来Table定位的时候可以使用的，特殊形状（如圆角）自适应伸缩Table，在标准页面里面还没有相应的实现方法。 其次，用Photoshop切碎的图，HTML里面可以用Table组合，XHTML难道用DIV一个一个去写？ 觉得以上两点就可以阻止一大批HTML好手进入XHTML，网站标准是不是需要在人性化一点，标准的同时是不是可以完善的更好一点？
